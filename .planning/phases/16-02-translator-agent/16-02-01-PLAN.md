  <action>
Create ai-engine/qa/translator.py with TranslatorAgent class that:

1. Takes QAContext as input (receives from QAOrchestrator)
2. Parses Java source files using existing patterns from agents/logic_translator.py
3. Queries RAG (HybridSearchEngine) for similar conversion patterns
4. Generates Bedrock JSON (behavior pack components - items, blocks, entities)
5. Generates TypeScript/JavaScript (Script API code)
6. Preserves code comments and documentation from original Java
7. Uses temperature=0 for deterministic LLM results
8. Validates output with validate_agent_output before returning
9. Stores translation results in context.validation_results['translator']

Key implementation details:
- Use existing LogicTranslatorAgent patterns for translation
- Integrate with HybridSearchEngine from search/ for RAG queries
- Handle context compression for large code blocks (chunk if >8000 tokens)
- Output both JSON files and TypeScript files based on component types
- Follow existing agent patterns from ai-engine/agents/

Example usage:
```python
from qa.translator import TranslatorAgent
from qa.context import QAContext

agent = TranslatorAgent()
result = agent.execute(context)  # Returns AgentOutput with translated code
```
