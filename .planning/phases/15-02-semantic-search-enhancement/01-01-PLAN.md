# Phase 15-02: Semantic Search Enhancement

**Phase ID**: 15-02
**Milestone**: v4.6 - RAG & Knowledge Enhancement
**Goal**: Hybrid search improvement, re-ranking, query expansion

## Goal
Enhance the semantic search capabilities with improved hybrid search, intelligent re-ranking, and advanced query expansion to maximize retrieval accuracy for Minecraft mod conversion queries.

## Context
- Previous: Phase 15-01 (Improved Document Indexing) - Just completed
- Backend: `backend/src/api/embeddings.py` - Embeddings API

## Implementation Requirements

### 1. Hybrid Search Engine

#### 1.1 Dense + Sparse Vector Combination
- Implement `HybridSearchEngine` in `ai-engine/search/hybrid_search.py`
- Combine dense (embedding) similarity with sparse (BM25) scores
- Configurable weighting (default: 0.7 dense, 0.3 sparse)
- Use reciprocal rank fusion (RRF) for score combination

#### 1.2 BM25 Integration
- Integrate rank-bm25 library for keyword search
- Build BM25 index from document chunks
- Update index on document additions/deletions
- Configure parameters: k1=1.5, b=0.75

#### 1.3 Search Result Merging
- Implement RRF algorithm: `score = sum(1/(k+rank))` where k=60
- Normalize scores from both methods before merging
- Support configurable top-k retrieval

### 2. Re-Ranking Pipeline

#### 2.1 Cross-Encoder Reranker
- Implement `CrossEncoderReranker` in `ai-engine/search/reranker.py`
- Use cross-encoder model: cross-encoder/ms-marco-MiniLM-L-6-v2
- Re-score top-k candidates (k=50) from hybrid search
- Support batch processing for efficiency

#### 2.2 Feature-Based Re-Ranking
- Add features: keyword overlap, recency, source authority
- Train simple LightGBM ranker on click-through data
- Fallback to cross-encoder if no training data

#### 2.3 Diversity Re-Ranking
- Implement MMR (Maximal Marginal Relevance) for diversity
- Balance relevance vs. diversity with lambda parameter
- Prevent duplicate source results in top-k

### 3. Query Expansion

#### 3.1 Synonym Expansion
- Build Minecraft modding synonym dictionary
- Expand queries with synonyms: "block" → "block, item, entity"
- Handle bidirectional synonyms (Java↔Bedrock terminology)

#### 3.2 Related Term Expansion
- Use embedding similarity to find related terms
- Expand with top-3 related terms from knowledge base
- Track expansion history for learning

#### 3.3 Context-Aware Expansion
- Consider conversation history for expansion
- Use entity recognition to identify key concepts
- Personalized expansion based on user history

### 4. API Integration

#### 4.1 Search API Enhancement
- Update `backend/src/api/search.py` with new parameters:
  - `use_reranker`: bool (default: true)
  - `use_hybrid`: bool (default: true)
  - `expand_query`: bool (default: true)
  - `diversity`: float (default: 0.3)

#### 4.2 Response Formatting
- Include reranking scores in response
- Provide explanation for result ordering
- Return expansion terms used
