  <action>
Create ai-engine/qa/fixer.py with FixerAgent class that:

1. Takes QAContext as input (receives from QAOrchestrator after ReviewerAgent)
2. Reads review results from context.validation_results['reviewer']
3. Attempts auto-fix for ESLint issues using eslint --fix
4. Fixes common JSON schema errors (missing keys, invalid values)
5. Corrects simple TypeScript type errors
6. Validates fixes don't introduce new issues by re-running review
7. Reports what was fixed and what couldn't be fixed
8. Uses validate_agent_output before returning
9. Stores fix results in context.validation_results['fixer']

Key implementation details:
- Follow same agent pattern as ReviewerAgent (ai-engine/qa/reviewer.py)
- Use subprocess to run eslint --fix and tsc --fix
- Apply JSON schema fixes based on Bedrock templates
- Track fix success rate
- Provide detailed fix report

Example usage:
```python
from qa.fixer import FixerAgent
from qa.context import QAContext

agent = FixerAgent()
result = agent.execute(context)  # Returns AgentOutput with fix results
```
