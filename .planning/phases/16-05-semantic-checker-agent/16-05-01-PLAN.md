  <name>Task 1: Create SemanticCheckerAgent class</name>
  <files>ai-engine/qa/semantic_checker.py</files>
  <action>
Create ai-engine/qa/semantic_checker.py with SemanticCheckerAgent class that:

1. Takes QAContext as input (receives from QAOrchestrator after FixerAgent)
2. Reads Java source from context.source_java_path
3. Reads generated Bedrock output from context.output_bedrock_path
4. Compares data flow graphs between Java and Bedrock:
   - Extract Java data flow (variable definitions, usage, mutations)
   - Extract Bedrock data flow (entity properties, events, callbacks)
   - Map Java variables to Bedrock equivalent properties
5. Analyzes control flow equivalence:
   - Extract Java method control flow (if/else, loops, switch)
   - Extract Bedrock event handler control flow
   - Check if logic paths are preserved
6. Verifies Script API method validity:
   - Check if called methods exist in known Script API surface
   - Flag usage of deprecated or non-existent API methods
7. Checks variable/type mappings:
   - Map Java types to Bedrock types (int→number, String→string, etc.)
   - Verify type compatibility in assignments and parameters
8. Generates semantic similarity score (0-100):
   - 100: Perfect semantic match
   - 70-99: Minor behavioral differences
   - 40-69: Moderate drift detected
   - 0-39: Major behavioral differences
9. Flags behavioral drift with detailed explanations
10. Handles edge cases and boundary conditions
11. Uses deterministic validation with explicit criteria
12. Validates output with validate_agent_output before returning
13. Stores semantic check results in context.validation_results['semantic']

Key implementation details:
- Follow same agent pattern as FixerAgent (ai-engine/qa/fixer.py)
- Parse Java using javalang for AST extraction
- Parse TypeScript/Bedrock JSON to extract structure
- Compare structural elements mathematically (not via LLM)
- Use weighted scoring: dataflow (30%), controlflow (25%), api_validity (25%), type_mappings (20%)
- Provide specific drift explanations with file/line references

Example usage:
```python
from qa.semantic_checker import SemanticCheckerAgent
from qa.context import QAContext

agent = SemanticCheckerAgent()
result = agent.execute(context)  # Returns AgentOutput with semantic analysis
```
