## Implementation Notes

### Learning Pipeline
1. User makes manual correction to converted code
2. System classifies the correction type
3. Pattern extracted if correction is reusable
4. Pattern added to database with confidence score
5. Future conversions use learned patterns

### Data Collection
- Track all manual edits during review
- Classify edits by impact (minor, moderate, major)
- Store correction context for pattern matching
- Aggregate patterns across all users (opt-in)
