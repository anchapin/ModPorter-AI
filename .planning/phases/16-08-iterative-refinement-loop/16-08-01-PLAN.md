  <action>
Add iterative refinement capability to QAOrchestrator:

1. Add new import: from qa.context import RefinementHistory
2. Add instance variables:
   - self.refinement_enabled: bool = True
   - self.max_refinement_iterations: int = 3
3. Add method `_detect_critical_issues(context: QAContext) -> List[Dict[str, Any]]`:
   - Check validation_results for any agent with errors or issues of severity CRITICAL
   - Return list of critical issues with agent_name, error message, suggestions
4. Add method `_build_refinement_prompt(context: QAContext, issues: List[Dict]) -> str`:
   - Include original source file path
   - Include list of critical issues with line numbers
   - Include suggestions from Reviewer/Tester
   - Return prompt string for Translator
5. Add method `run_with_refinement(context: QAContext) -> QAContext`:
   - Run initial QA pipeline (parallel or sequential based on config)
   - Calculate initial quality score from validation_results
   - Loop up to max_refinement_iterations:
     - Detect critical issues
     - If no critical issues, break loop
     - Build refinement prompt with issues
     - Modify context for refinement (update metadata with prompt)
     - Re-run translator agent with refinement context
     - Re-run remaining agents (reviewer, tester, semantic_checker)
     - Calculate new quality score
     - Record iteration in refinement_history
     - If score improvement < 5%, break loop
   - Set context.refinement_completed = True
   - Return context
6. Add method `_calculate_quality_score(validation_results: Dict) -> float`:
   - Extract score from each agent's validation_result
   - Return weighted average (or 0 if no scores)
  </action>
