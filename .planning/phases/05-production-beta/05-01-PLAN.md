    build:
      context: ./backend
      dockerfile: Dockerfile
    environment:
      - ENVIRONMENT=production

      retries: 3

- [ ] Jaeger tracing enabled

**Alert Rules**:
```yaml
# alert_rules.yml
groups:
  - name: modporter
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.05
        for: 5m
        annotations:
          summary: "High error rate detected"
      
      - alert: HighLatency
        expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 1
        for: 10m
        annotations:
          summary: "High latency detected (p95 > 1s)"
      
      - alert: ServiceDown
        expr: up{job=~"backend|frontend|ai-engine"} == 0
        for: 2m
        annotations:
          summary: "Service {{ $labels.job }} is down"
```

### Task 2.1.5: Backup System

**Type**: Implementation  
**Effort**: Medium (3 hours)  
**Dependencies**: Task 2.1.2  
**Verification**: Backups running successfully

**Description**: Configure automated backups.

**Acceptance Criteria**:
- [ ] PostgreSQL backup script created
- [ ] Daily backup schedule configured (cron)
- [ ] Backups stored off-server (S3, Backblaze)
- [ ] Backup retention policy (30 days)
- [ ] Restore procedure documented and tested

**Backup Script**:
```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups"
S3_BUCKET="modporter-backups"

# Database backup
docker-compose exec -T postgres pg_dump -U modporter modporter > ${BACKUP_DIR}/db_${DATE}.sql

# Compress
gzip ${BACKUP_DIR}/db_${DATE}.sql

# Upload to S3
aws s3 cp ${BACKUP_DIR}/db_${DATE}.sql.gz s3://${S3_BUCKET}/

# Cleanup old backups (keep 30 days)
find ${BACKUP_DIR} -name "db_*.sql.gz" -mtime +30 -delete
```
