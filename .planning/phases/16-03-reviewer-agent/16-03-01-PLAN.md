  <action>
Create ai-engine/qa/reviewer.py with ReviewerAgent class that:

1. Takes QAContext as input (receives from QAOrchestrator after TranslatorAgent)
2. Runs ESLint/TSLint on generated TypeScript files
3. Validates JSON against Bedrock schemas (blocks, items, entities)
4. Checks TypeScript types via tsc compilation
5. Verifies Script API method usage against known API surface
6. Flags issues with line numbers and severity levels (error, warning, info)
7. Provides auto-fix suggestions for common issues
8. Generates quality score (0-100) based on validation results
9. Uses deterministic validation with explicit rubrics
10. Validates output with validate_agent_output before returning
11. Stores review results in context.validation_results['reviewer']

Key implementation details:
- Follow same agent pattern as TranslatorAgent (ai-engine/qa/translator.py)
- Use subprocess to run eslint/tsc commands
- For JSON schema validation, load Bedrock template schemas from agents/logic_translator.py
- Quality score calculation: 100 - (errors * 10) - (warnings * 3) - (info * 1), min 0
- Handle missing tools gracefully (skip check if tool unavailable)
- Output quality report with issue breakdown

Example usage:
```python
from qa.reviewer import ReviewerAgent
from qa.context import QAContext

agent = ReviewerAgent()
result = agent.execute(context)  # Returns AgentOutput with review results
```
