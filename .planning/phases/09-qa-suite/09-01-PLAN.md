# Plan: Phase 09-01 - Automated Conversion Validation

## Overview
Implement automated validation for Java source and Bedrock output, covering syntax, structure, semantics, and cross-references.

## Requirements Covered
- REQ-09-01-001: Java Syntax Validation (Critical)
- REQ-09-01-002: Bedrock Syntax Validation (Critical)
- REQ-09-01-003: Structure Validation (High)
- REQ-09-01-004: Semantic Validation (High)
- REQ-09-01-005: Cross-Reference Validation (Medium)

## Tasks

### Task 09-01-01: Java Syntax Validation
<action>
Create JavaSyntaxValidator in ai-engine/agents/validation_agent.py that:
- Uses javalang to parse Java source files
- Returns parsing errors with line/column information
- Handles multiple file inputs
</action>
<verify>
- Unit test parses valid Java files without errors
- Unit test correctly identifies and reports parsing errors with line numbers
</verify>
<done>
- JavaSyntaxValidator class implemented with javalang integration
- Error reporting includes line/column position
</done>

### Task 09-01-02: Bedrock Syntax Validation
<action>
Create BedrockSyntaxValidator that:
- Validates JSON files using Python json module
- Validates JavaScript using esprima or similar
- Checks manifest.json structure
</action>
<verify>
- Valid JSON files pass validation
- Invalid JSON reports parse errors with position
- JavaScript syntax errors are detected
</verify>
<done>
- BedrockSyntaxValidator class implemented
- JSON validation covers manifest.json and pack files
- JavaScript validation for behavior files
</done>

### Task 09-01-03: Structure Validation
<action>
Create StructureValidator that:
- Checks required pack files (manifest.json, pack_icon.png)
- Validates behavior pack directories (functions, recipes, loot_tables)
- Validates resource pack directories (textures, models, sounds)
</action>
<verify>
- Complete packs pass validation
- Missing required files are flagged
- Directory structure is verified
</verify>
<done>
- StructureValidator implemented
- Checks all required Bedrock pack structure
</done>

### Task 09-01-04: Semantic Validation
<action>
Create SemanticValidator that:
- Verifies Java class references resolve correctly
- Checks method calls match Bedrock API equivalents
- Validates entity/component mappings
</action>
<verify>
- Known mappings are validated
- Invalid mappings are flagged with suggestions
- Cross-language references are checked
</verify>
<done>
- SemanticValidator class implemented
- Uses existing mapping data from conversion system
</done>

### Task 09-01-05: Cross-Reference Validation
<action>
Create CrossReferenceValidator that:
- Validates JSON references (loot tables, recipes)
- Checks texture/model references exist
- Verifies namespace consistency
</action>
<verify>
- Broken references are detected
- References to non-existent files are flagged
- Namespace consistency is checked
</verify>
<done>
- CrossReferenceValidator implemented
- Scans all JSON for reference validation
</done>

### Task 09-01-06: Validation Pipeline Integration
<action>
Create ValidationPipeline in backend/src/api/validation.py that:
- Runs all validators in sequence
- Aggregates results
- Provides unified API endpoint
</action>
<verify>
- All validators run on conversion output
- Results are aggregated with severity
- API returns structured validation report
</verify>
<done>
- /api/v1/validate endpoint implemented
- Returns JSON with all validation results
</done>
