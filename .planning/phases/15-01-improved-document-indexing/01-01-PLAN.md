# Phase 15-01: Improved Document Indexing

**Phase ID**: 15-01  
**Milestone**: v4.6 - RAG & Knowledge Enhancement  
**Goal**: Smart chunking, metadata extraction, hierarchical indexing

## Goal
Implement intelligent document indexing with semantic-aware chunking, rich metadata extraction, and hierarchical index structures to improve RAG retrieval accuracy and performance.

## Context
- Previous: Phase 14-07 (Sealed Classes) - 170+ tests passing
- Backend: `backend/src/api/embeddings.py` - Embeddings API
- AI Engine: `ai-engine/search/` - hybrid search, reranking, query expansion
- Database: PostgreSQL + pgvector (VECTOR(1536))
- Embedding Model: sentence-transformers/all-MiniLM-L6-v2

## Success Criteria

### Must Have
1. Smart chunking respects semantic boundaries (paragraphs, sections, code blocks)
2. Rich metadata extraction (source, date, type, tags, headings)
3. Hierarchical index structure (document → section → chunk)
4. Backward compatible with existing embeddings API
5. Indexing throughput ≥ 100 chunks/second for typical documents

### Verification
- Unit tests for chunking strategies (≥90% coverage)
- Integration tests for metadata extraction
- Performance benchmarks for indexing speed
- Existing API tests pass without modification

## Implementation Requirements

### 1. Smart Chunking Module

#### 1.1 ChunkingStrategy Enum & Factory
- Create `ai-engine/indexing/chunking_strategies.py`
- Implement strategies: `FixedSizeChunking`, `SemanticChunking`, `RecursiveChunking`
- Factory pattern for strategy selection

#### 1.2 SemanticChunking Implementation
- Detect paragraph boundaries using whitespace/line break patterns
- Respect code block boundaries (``` fences, indentation)
- Maintain heading hierarchy context
- Configurable chunk size (default: 512 tokens) with overlap (default: 50 tokens)

#### 1.3 RecursiveChunking Implementation  
- Split by hierarchical separators: headings (#, ##, ###), paragraphs, sentences
- Fallback to smaller granularity if chunk exceeds max size
- Preserve context references to parent chunks

#### 1.4 Chunk Metadata Builder
- Extract heading context (parent headings)
- Calculate chunk position (index, total chunks)
- Generate chunk-specific hash for deduplication

### 2. Metadata Extraction

#### 2.1 DocumentMetadataExtractor
- Extract from document headers: title, author, date, description
- Detect document type (markdown, code, plain text, PDF-like)
- Generate tags from content analysis

#### 2.2 ChunkMetadata Schema
```python
class ChunkMetadata(BaseModel):
    source_document_id: str
    chunk_index: int
    total_chunks: int
    heading_context: List[str]  # Parent headings
    document_type: str
    extracted_tags: List[str]
    original_heading: Optional[str]
    char_start: int
    char_end: int
```

#### 2.3 Database Model Updates
- Add `metadata` JSONB column to `DocumentEmbedding` table
- Add `parent_document_id` for hierarchical relationships
- Add `chunk_index` for ordering

### 3. Hierarchical Indexing

#### 3.1 Document Hierarchy Structure
```
Document (top level)
  └── Sections (mid level) 
        └── Chunks (leaf level with embeddings)
```

#### 3.2 ParentReferenceField
- Add `parent_id` (UUID) to link chunks to parent document/section
- Add `hierarchy_level` (int): 0=document, 1=section, 2=chunk

#### 3.3 Hierarchical Retrieval
- Query parent document to get all related chunks
- Support section-level search with child chunk expansion

### 4. API Extensions (Backward Compatible)

#### 4.1 New Endpoints
- `POST /embeddings/index-document` - Index document with smart chunking
- `GET /embeddings/documents/{doc_id}` - Get document with all chunks
- `GET /embeddings/documents/{doc_id}/chunks` - Get document chunks

#### 4.2 Request/Response Models
```python
class IndexDocumentRequest(BaseModel):
    content: str
    source: str
    metadata: Optional[dict]  # Optional user-provided metadata
    chunking_strategy: str = "semantic"  # "fixed", "semantic", "recursive"

class IndexDocumentResponse(BaseModel):
    document_id: str
    chunks_created: int
    metadata: dict
```
