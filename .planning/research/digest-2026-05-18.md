### Paper Details

#### 1. Beyond Translation Accuracy: Addressing False Failures in LLM-Based Code Translation
- **ArXiv**: https://arxiv.org/abs/2605.02195
- **Date**: May 4–8, 2026
- **Relevance**: 85%
- **Summary**: Large-scale study across 6,164 translations on 5 PLs (C, C++, Java, Python, Go) and 3 benchmarks found that many reported failures are pipeline-induced, not model-induced — caused by improper compilation flags, missing library links, and unconfigured runtime environments. Distinguishes pipeline-induced (affects all models) from model-dependent failures.
- **PortKit Applicability**: PortKit's eval harness may have the same classes of false failures — Bedrock API shim misconfiguration, missing NBT/resource links, unenforced Bedrock runtime constraints. Current pass rate could be artificially pessimistic or optimistic. Critical for MMSD fine-tuning signal quality.

#### 2. CodePivot: Bootstrapping Multilingual Transpilation via RL without Parallel Corpora
- **ArXiv**: https://arxiv.org/abs/2604.18027
- **Date**: April 2026
- **Relevance**: 85%
- **Summary**: Trains 7B transpilation model using Python as pivot IR, no parallel corpus required. Aggressive-Partial-Functional (APF) reward gives partial credit for partial translations. Outperforms DeepSeek-R1 and Qwen3-235B on transpilation tasks.
- **PortKit Applicability**: MMSD has 1,400 pairs — limited for long-tail mod complexity. Pivot-IR approach enables training on abundant unpaired Java + Bedrock code. APF reward directly complements #1269 Minecraft reward model — APF handles partial credit, #1269 handles domain idiomaticity.

#### 3. Iterative Audit Convergence in LLM-Managed Multi-Agent Systems
- **ArXiv**: https://arxiv.org/abs/2605.12280
- **Date**: May 12, 2026
- **Relevance**: 80%
- **Summary**: 7-lane production pipeline (AEGIS) audited by Claude sub-agents using checklist-driven walkthrough. 9 rounds surfaced 51 prompt-spec consistency defects. Single-file review missed entire defect classes only visible during cross-file scope expansion.
- **PortKit Applicability**: PortKit's LangGraph pipeline has multiple interdependent agent lanes with data contracts flowing across boundaries. Lane-level prompt inconsistencies silently degrade complex conversion quality. Iterative agent-audit could surface defect classes invisible to standard review.

### 65–79% Relevance (Horizon Notes)

#### 4. Evaluating LLM-Generated Code: A Benchmark and Developer Study
- **ArXiv**: https://arxiv.org/abs/2605.09059
- **Date**: May 9, 2026
- **Relevance**: 75%
- **Summary**: Tree-fold evaluation methodology: correctness benchmark + code quality verification + developer survey. Applied to GPT-4.1, DeepSeek-V3, Claude Opus 4. Developer reviews surface production-readiness issues invisible to automated correctness benchmarks.
- **PortKit Horizon**: PortKit's eval is currently binary pass/fail. Tree-fold approach (correctness + Bedrock idiomaticity + modder-readiness perception) would give richer signal. Relevant once Phase 0.3 QA is stable and we need to push toward production quality benchmarking.

#### 5. LLM Translation of Compiler Intermediate Representations (IRIS-14B)
- **ArXiv**: https://arxiv.org/abs/2605.08247
- **Date**: May 2026
- **Relevance**: 65%
- **Summary**: 14B model fine-tuned for GIMPLE-to-LLVM IR translation. Frames LLMs as interoperability layers in hybrid neuro-symbolic compiler architectures. Outperforms models up to 1T params on IR-to-IR tasks.
- **PortKit Horizon**: Focused on compiler IR (too low-level for PortKit's use case), but the neuro-symbolic interoperability layer concept has structural similarity to PortKit's Java AST → Bedrock Scripting API translation. Horizon only.
