# Phase 13-02: Lambda Expression Support

**Phase**: 13-02  
**Goal**: Implement Java lambda expression to Bedrock JavaScript conversion support  
**Status**: Ready for Implementation  
**Complexity**: Medium-High  
**Estimated Time**: 4-5 hours

---

## Implementation Tasks

### Task 1: Create Lambda Expression Detector

**File**: `ai-engine/utils/lambda_detector.py` (NEW)

Create a utility module to detect and analyze lambda expressions in Java AST:

- Detect lambda expression nodes in AST (both expression lambdas and block lambdas)
- Extract lambda parameters (with and without type annotations)
- Identify lambda body (expression vs block)
- Handle captured variables from enclosing scope
- Detect method references (instance, static, super, constructor)

**Test**: `tests/test_lambda.py::test_lambda_detection`

### Task 2: Create Lambda to Function Mapper

**File**: `ai-engine/utils/lambda_to_function_mapper.py` (NEW)

Map Java lambda expressions to Bedrock-compatible JavaScript functions:

- Convert lambda syntax to arrow functions or regular functions
- Handle parameter type inference
- Map captured variables to closure scope
- Handle method references to function.bind() or arrow functions
- Generate unique names for converted lambda functions

**Test**: `tests/test_lambda.py::test_lambda_to_function_mapping`

### Task 3: Create Lambda Type Inference

**File**: `ai-engine/utils/lambda_type_inference.py` (NEW)

Infer functional interface types from lambda expressions:

- Detect target functional interface from context (Predicate, Function, Consumer, Supplier, etc.)
- Infer return type from lambda body
- Validate lambda compatibility with target interface
- Handle void-compatible lambdas (Consumer patterns)

**Test**: `tests/test_lambda.py::test_lambda_type_inference`

### Task 4: Extend Java Analyzer

**File**: `ai-engine/agents/java_analyzer.py`

Enhance the Java analyzer to detect lambda expressions:

- Add `detect_lambda_expressions()` method
- Extract lambda metadata (parameters, body, captured variables)
- Identify lambda usage contexts (assignments, method arguments, returns)
- Store lambda info in analysis results

**Test**: `tests/test_lambda.py::test_java_analyzer_lambda`

### Task 5: Extend Logic Translator

**File**: `ai-engine/agents/logic_translator.py`

Update translator to convert lambda expressions:

- Add lambda expression handling in translation pass
- Convert lambdas to appropriate JavaScript functions
- Handle nested lambdas and lambda chains
- Manage scope and closure conversion

**Test**: `tests/test_lambda.py::test_logic_translator_lambda`

### Task 6: Create Integration Tests

**File**: `tests/test_lambda.py`

Comprehensive tests for the complete lambda conversion pipeline:

- Test simple lambda expressions (x -> x + 1)
- Test multi-parameter lambdas ((x, y) -> x + y)
- Test block lambdas (x -> { return x * 2; })
- Test method references (String::length, ArrayList::new)
- Test captured variables
- Test lambda in streams/collections
- Test type inference

**Target**: 15+ passing tests
